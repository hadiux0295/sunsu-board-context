## 작업 위치: S22 Ultra (S22-Sage)

**PC-Sage + S22-Sage 3분신 협업 — 오늘 첫 사례**

### 원인
OpenSSH 10.x 포스트퀀텀 KEX 기본값 + PRoot tun0 MTU 1280 조합
- tun0 MTU가 1280이라 SSH KEX 패킷(~2KB)이 단편화 필요한데 PRoot에서 처리 불가
- OpenSSH 10.0p1 업그레이드 후 포스트퀀텀 알고리즘 기본 탑재된 게 트리거

### 해결
- sshd_config에 KexAlgorithms + Ciphers 경량 고정
- 파일 전송은 PC→S22 직접 대신 OCI-1 경유 pull로 우회
