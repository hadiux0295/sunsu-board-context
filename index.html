# [Zeroclaw 인프라 혁신 보고서] 설정 의존성 극복 및 자가복구(Self-Healing) 시스템 구축 완료 🚀

**작성자:** Hun & AI Team (Lumi, Sunsu, Sage)  
**일자:** 2026-04-05  
**목표:** Zeroclaw 런타임 환경의 불안정성 해소 및 완벽한 상태 통제 시스템 구축

---

### 0. 오늘의 발견 (Origin Story)
Zeroclaw 자율복구 테스트 중 버그 3개를 발견한 에이전트의 정체가 사실 OpenRouter를 경유한 `claude-sonnet-4.6`이었음이 사후 확인됨.  
이미지 업데이트로 인해 기본값이 해당 모델로 변경된 상태였고, `settings.json` 부재로 인해 조용히 작동 중이었음. 당시 Sage는 자신의 활약을 모른 채 시스템을 칭찬했고, Lumi가 이를 인증하는 촌극(자화자찬 사태)이 발생. 🎭  
👉 **이 사건이 설정 드리프트의 무서움을 증명하며, `settings.json` 고정과 `env_sync.py` 도입의 직접적인 계기가 됨.**

### 1. 배경 및 문제점 (Before)
기존 Zeroclaw 인프라는 재시작, 이미지 업데이트, 또는 예기치 않은 오류 발생 시 런타임 환경설정(`.env`)이 외부 기본값으로 쉽게 오염되는 치명적인 취약점이 있었습니다.
- **설정의 가변성:** 설정이 일원화되지 않아 시간이 지날수록 기준점과 실제 상태가 벌어지는 현상 발생.
- **조용한 실패(Silent Failure):** 환경 변수 누락이나 오염이 발생해도 시스템이 명확한 에러를 뿜지 않고 오작동하여 디버깅 난이도를 급상승시킴.
- **상태 의존적 시스템:** 환경이 망가지면 사람이 직접 개입하여 복구해야만 하는 구조적 한계.

### 2. 해결책: 구조적 혁신 (After)
버그를 단순히 수정하는 것을 넘어, **"환경이 스스로 일관성을 유지하는 시스템"**으로 아키텍처를 전면 개편했습니다.

- **🔒 Single Source of Truth (단일 진실원):** `/a0/usr/.env.master`를 읽기 전용 절대 기준으로 설정하고, 실제 실행 환경인 `.env`는 언제든 재생성될 수 있는 소모성 파일로 전환.
- **🧩 허용된 변화 vs 금지된 변화 분리 정책:** 마스터에 정의되지 않은 런타임 추가 환경 변수는 삭제하지 않고 유지하며, 오직 명시적으로 금지된 값(forbidden)과 필수 키(required)에 대해서만 강제 교정을 수행.
- **🔄 Self-Healing 루프 구조:** 단순 1회성 검증으로 끝나지 않고, `검증 → 수정 → 재검증`의 루프를 통해 최종적으로 완전한 일관 상태에 도달할 때까지 반복 수행.
- **🔑 해시 서명(Signature) 검증 도입:** `.env` 최상단에 `# MASTER_SIGNATURE=...` 형태로 마스터 파일의 SHA-256 해시값을 강제 주입하여, 현재 환경이 단순히 '비슷한 상태'가 아니라 **'마스터와 완전히 동일한 상태'임을 수학적으로 보증**.

### 3. 제거된 실패 패턴 (Failure Modes Eliminated)
본 시스템 도입으로 다음과 같은 고질적인 장애 유형이 영구적으로 제거되었습니다. 이는 단순한 안정성 향상이 아닌, 시스템 신뢰성(Reliability)의 구조적 확보를 의미합니다.
- 설정 드리프트 (Configuration Drift)
- 외부 기본값 오염 (예: Claude 기본 설정 재유입)
- 환경 변수 누락으로 인한 비정상 동작
- Silent Failure (명시적 에러 없이 기능만 실패하는 상태)
- 재시작 시 상태 불일치

### 5. 다음 단계 및 알려진 리스크 (Next Steps & Risks)

1. **Secret 완전 분리 (Zero Hardcoding):**
   - ⚠️ **알려진 리스크:** 현재 `MCP_TOKEN`은 `A0_PERSISTENT_RUNTIME_ID` 기반 고정값이나 `.env.master`에 하드코딩된 상태. Runtime ID 변경 시 전체 MCP 연결이 끊어질 위험이 존재.
   - **대응:** 해당 키 등 민감 정보는 마스터에서 제거하고, 런타임 시 외부 Secret Source에서 동적으로 주입하도록 개편 (1순위 과제).
2. **부팅 시 Self-Healing Hook 완전 자동화:**
   - `env_guard.sh`를 zeroclaw entrypoint에 연결하여, 컨테이너 시작 시 무조건 환경 검증 및 교정 수행.
3. **환경 변경 로그 시스템 도입 (Drift Logging):**
   - 어떤 키가 언제, 어떻게 수정되었는지 기록하여 장기적으로 시스템 이상 패턴 분석이 가능하도록 확장.
