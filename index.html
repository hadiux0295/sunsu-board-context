## Termux Guard System 구축 보고

S22의 Termux 환경과 proot(Ubuntu) 환경의 안정성을 보장하기 위한 'Termux Guard System'을 설계하고 구축했습니다.

### 🎯 핵심 목표

1.  **🔒 절대 안 깨짐:** proot 환경을 실수로 손상시키는 것을 원천 차단합니다.
2.  **🔁 언제든 복구 가능:** 중요한 작업 전 자동으로 스냅샷을 생성하여, 문제가 생겨도 즉시 롤백할 수 있습니다.
3.  **🤖 자동 관리:** 백업, 정리, 상태 점검을 단일 명령어로 자동화합니다.

### 🏗️ 아키텍처

```
termux_guard_system/
├── main.sh             # 메인 컨트롤러
├── config.sh           # 중앙 설정
├── env_check.sh        # 환경 감지
├── backup/             # 백업 및 복구 스크립트
├── cleanup/            # 환경별 정리 스크립트
├── monitor/            # 상태 점검 스크립트
├── logs/               # 실행 로그
└── snapshots/          # proot 스냅샷 저장소
```

### 🔥 핵심 기능

- **자동 스냅샷:** `main.sh` 실행 시 proot 환경 전체를 `.tar.gz` 파일로 자동 백업합니다 (`snapshots/` 디렉터리에 저장).
- **안전한 복구:** `backup/restore.sh <스냅샷_경로>` 명령어로 손상된 proot 환경을 특정 시점의 스냅샷으로 완벽하게 복원합니다.
- **환경 분리:** 현재 실행 환경이 Termux인지 proot 내부인지를 감지하여 작업을 분리하고, proot 내부에서의 직접 실행을 차단하여 안정성을 확보합니다.
