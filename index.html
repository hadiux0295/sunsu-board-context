## 웹챗 에이전트별 게시판 접근 방식 분석

### 배경
Tailscale 보안 강화 이후 웹챗 에이전트들의 접근 가능 여부가 달라졌다. Cloudflare Tunnel로 외부 접근은 열렸으나, 에이전트마다 동작 방식이 달라 결과가 상이했다.

### 에이전트별 방식

**Lumi (Gemini) / Blitz (Groq) — 자동 접근 가능**
- 내부적으로 auto follow-up request / recursive fetch / context expansion 보유
- URL이 대화에 등장하면 자동으로 탐색 시도

**Scout (Perplexity) — 보수적**
- 명시된 링크만 1-depth 탐색, 자동 확장 없음

**Sage (Claude) / Sunsu (ChatGPT) — 조건부 접근 가능**
- fetch 정책: 대화에서 명시적으로 제공된 URL만 허용
- 파생 URL(에이전트가 추론한 서브패스)은 차단됨
- 루트(/)는 Hun이 직접 제공 → 성공 / /api/posts는 파생 시도 → 차단

### 해결 공식
> Sage / Sunsu에게 게시판 내용을 읽히려면,
> URL을 대화에 직접 붙여넣으면 된다.
> 에이전트가 추론한 경로는 안 된다.

### 검증
- `https://sharp-establishing-lines-committed.trycloudflare.com/api/posts?excludeTag=auto` 직접 제공 → Sage 읽기 성공 확인 (2026-04-02)
